# Contributing to Sentinel

Thanks for contributing to Sentinel.

Sentinel is a community-driven project. We want contributions that improve security, accessibility, reliability, documentation, developer experience, and long-term maintainability.

## Before you start

Please:
1. Read the `README`, `LICENSE`, `SECURITY.md`, and this file.
2. Search existing issues and pull requests before opening a new one.
3. Prefer small, focused changes over broad refactors unless discussed first.
4. For security-sensitive changes, be extra clear about risks, trade-offs, and testing.

## Ways to contribute

You can help by contributing:
- bug fixes
- documentation improvements
- tests
- security hardening
- performance improvements
- examples
- accessibility and DX improvements
- issue triage and reproduction steps

## Issues

When opening an issue, include:
- what you expected to happen
- what actually happened
- steps to reproduce
- environment details
- logs, screenshots, or traces when relevant

Please keep reports specific and reproducible.

## Pull requests

### General expectations

Pull requests should:
- be scoped to a clear goal
- include tests when appropriate
- avoid unrelated changes
- update docs when behavior changes
- explain the reasoning behind the change, not just the code

### PR description checklist

Please include:
- summary of the change
- motivation
- screenshots or sample requests if UI/API behavior changed
- testing notes
- known limitations or follow-up work

## Coding principles

Contributions should aim for:
- clarity over cleverness
- secure defaults
- minimal surprise in public interfaces
- maintainable abstractions
- backwards compatibility when possible

## Security-related contributions

Do not open public issues for undisclosed vulnerabilities.
Please follow the private reporting instructions in `SECURITY.md`.

## License for contributions

By submitting a contribution, you agree that your contribution will be licensed under the same license as this project.

## Community standards

By participating in this project, you agree to follow the `CODE_OF_CONDUCT.md`.

Thanks for helping keep Sentinel open, useful, and trustworthy.
