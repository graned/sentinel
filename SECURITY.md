# Security Policy

Sentinel is a security-sensitive project. Responsible disclosure is appreciated.

## Reporting a vulnerability

Please do **not** report security vulnerabilities through public GitHub issues.

Instead, report them privately to the maintainer through the repository contact channel or security contact listed in the repository profile.

When reporting a vulnerability, please include:
- a clear description of the issue
- affected components or versions
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigations, if known

## What to expect

Best effort process:
1. acknowledgement of receipt
2. initial triage
3. validation and severity assessment
4. fix planning and release coordination
5. public disclosure after a fix is available, when appropriate

## Disclosure guidelines

Please allow reasonable time for validation and remediation before public disclosure.

## Supported versions

At minimum, the latest maintained release branch should be assumed supported unless the repository states otherwise.

## Security principles

Sentinel aims to follow these principles:
- least privilege
- secure defaults
- defense in depth
- transparent disclosure after remediation
- reproducible and reviewable changes
